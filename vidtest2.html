HTML5 tests - video

This page tests the <video> tag.

On the mobile side iPhone supports H.264, and that’s it.

Test cases

Note the syntax; IE and Firefox need the source and type.

<video controls>
	<source src="../videos/big_buck_bunny.mp4" type="video/mp4">
	<p>Your browser does not support H.264/MP4.</p>
</video>

Fallback content only supported when the browser does not support the <video> tag at all. Bloody useless; what you’d want is to switch to fallback when, say, you only have WebM available and the browser doesn’t support that codec.
